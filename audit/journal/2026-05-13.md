## Mechanisms understood

- Demo is the FullCalendar-style options shape (`headerToolbar` left/center/right, `initialView`, `initialDate`, `views`, `editable`, `selectable`, `resources`, `initialEvents`, `eventResize`, `select`, `eventClick`, `eventsSet`, `viewDidMount`, `locales`/`locale`). chronix needs the same options surface to drive R2 parity.
- Six timeline scales (`day/week/month/season/halfYear/year`) are driven by built-in view presets — `views: {...}` block in the demo is commented out, so chronix must replicate the defaults, not the override paths.
- Resource area is a grouped-column tree (`resourceAreaColumns` with `group: true` on header rows); width is a percentage (`'20%'`).
- Per-event `progress` is a structured object: `{ value (0-100), backgroundColor, textColor, textFormat: '- {value}% xxx' }` rendered inside the bar.
- `eventStyleCallbacks` + `eventProcessCallback` are post-render hooks taking an EventContentArg-shaped argument (`isSelected`, `isDragging`, `event.extendedProps.*`, `backgroundColor`). Brightness-based auto-contrast in the demo at DemoApp.vue:1466-1495 is empirical; we must match the same rounding curve in goldens.
- Dependency lines: per-event `extendedProps.dependencies: string[]` declares predecessors; `onLine(line)` mutates `markerType`/`customMarker` per line at render time. `useLineEventColor` makes lines inherit start-event color; `lineStyle: 'square' | 'smooth'`.
- Today line: object `{ color, width, style: 'solid'|'dashed'|'dotted'|'double'|'dashed-dot', tooltip }` or `false`.
- **Date-anchoring**: all events go through `formatDateTime(addDays(new Date(), N), hour)` — every event is computed relative to `new Date()` at module load. For goldens this is the largest threat: identical demo bundle produces different DOM on different days. Phase 0 capture must freeze the clock (Playwright `page.clock.install({ time: '2026-05-13T08:00:00Z' })` set before navigation, or stub `Date` via init script) so the golden PNG/JSON is reproducible across runs and machines.
